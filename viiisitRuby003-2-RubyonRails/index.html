建立 Route

首先，要把路徑建立起來，才能通過這些路徑來對應各個頁面。
因為 Route 是整個網站連結各個頁面的相關路徑，當使用者連上網站時，Rails 會解析使用者所輸入的網址及參數，去找到該負責處理 Controller 與 Action。

before_action :基礎知識

再次複習一下，在 Viiisit Ruby on Rails - 先備知識! 文章提及的小觀念：

  • 透過 Rails 的路徑設定，可以快速地建立符合 RESTful 概念的路由，
    並使應用程序遵循 RESTful 架構來處理資源的 CRUD 操作。

  • get, post, patch, delete 是先前在 CRUD 提過的路徑對 HTTP 的請求方法，
    這些 HTTP 方法與 CRUD 操作關係如下：

    • GET 對應 Read 操作，用於獲取資料。
    • POST 對應 Create 操作，用於創建新的資料。
    • PATCH 對應 Update 操作，用於部分更新資料。
    • DELETE 對應 Delete 操作，用於刪除資料。

  • to: controller#action (後面接的是對應的 controller 與 action)

  • 在建立這些 route, controller, action 之間的連結後，交由 view 去呈現畫面。

在 routes.rb 這個檔案裡，把所需要的路徑先建立起來！

Shallow Nesting 是在設計 Rails 路徑時的一種方式，
目的是將深度嵌套的路徑結構簡化，使路徑設定更加直觀和易於管理。

建立 Controller 與定義 Action

在先前介紹路徑時，會同時命名 controller 與 action。
依照 Rails 的慣例，controller 的名字通常跟路徑一致。
這種命名幫助我們更好地組織與管理 controller 與 action，程式碼也更易於理解和維護。
同时也符合 Rails 的約定優於配置（Convention over Configuration）的原則。

透過 rails generate controller Articles or rails g controller Articles 指令，
生成叫做 Articles 的 controller！

Remark:
controller 建立， rails generate controller 自定義名字 -> rails g controller 自定義名字
controller 移除， rails destroy controller 自定義名字 -> rails d controller 自定義名字

剛剛的指令可以幫我們生成一些所需要的檔案，
而上方路徑建立的 Action 就會被定義在 controller 裡：

建立 View

當前面的 Route, Controller 與 Action 都建立好後，就需要建立 View 呈現畫面！
需要手動新增檔案：action 名稱.html.erb，建立 view
（以 index 為例） index.html.erb

app/views/articles/index.html.erb

特別以 new.html.erb 來說明 form_with 可以生成所需要的表單並自動生成所需要的表單並包含 CSRF token，保護你的應用免受 CSRF 攻擊。

Remark:
在 Rails 中，POST 行為具有保護機制。
如果在設定好的動作中重新整理頁面，並且缺少 view，你會注意到與之前不同的錯誤訊息：ActionController::InvalidAuthenticityToken in ArticlesController#create
這時候，你需要設定 CSRF token 保護機制，以便在提交表單後能夠通過！
_有種情況下不需要設定 CSRF token：當你需要跟第三方金流做連線時，需要把這個保護機制關閉，這樣才能互相聯繫。 非同步交易時，當完成之後，會透過 Notify-URL 去通知對方_

app/views/articles/new.html.erb
