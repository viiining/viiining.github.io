    為什麼 Model 需要關聯性？

    我們在一個專案裡，一定不只有一個 Model，會同時存在許多 Model，
    在 Rails 裡，我們可以為這些 Model 做許多關聯，大致上歸納出以下優點：

    Model 需要關聯性是為了更好地組織和管理資料，以及實現資料間的關聯和連結。

    • 資料的結構化：通常一個應用程式涉及多個資料，彼此之間可能存在著很複雜的關係。
      使用 Model 的關聯性可以將這些資料的關係結構化，使得資料庫中的資料更容易理解和管理。

    • 整合資料：在真實的應用程式中，資料往往存在著相互關聯的情況。例如：一個使用者有多個訂單。
      使用 Model 的關聯性可以將這些相關聯的資料整合在一起，使得查詢和操作資料更加方便。

    • 簡化資料存取：使用 Model 的關聯性可以讓你在查詢資料時不需要手動撰寫複雜的 SQL 語句，而是使用 Rails 提供的簡單方法即可。這樣可以節省開發時間，並減少出錯的可能性。

    • 維護性和擴展性：當應用程式需要新增新的資料實體或調整資料結構時，使用 Model 的關聯性可以輕鬆地進行修改而不影響其他部分的程式碼。這增加了程式的維護性和擴展性。

    關聯種類

    Rails 支援以下六種關聯：

    • belongs_to
    • has_one
    • has_many
    • has_many :through
    • has_one :through
    • has_and_belongs_to_many

    Model 關聯性

    在 Rails 中，Model 的關聯性是指不同 Model 之間的連結與關係。
    這種關聯性可以在資料庫層面建立表格之間的連結，使得資料之間可以相互關聯和互動，
    進而簡化資料的查詢和操作。

    Rails 支援多種不同類型的 Model 關聯性：

    1. 一對一關聯（One-to-One Association）：兩個 Model 之間只有一個對應關係。
      例如：一個使用者只有一個個人檔案。

    2. 一對多關聯（One-to-Many Association）：一個 Model 可以對應到多個其他 Model。
      例如：一個使用者有多篇文章。

    3. 多對多關聯（Many-to-Many Association）：兩個 Model 之間可以相互對應多個實例。
      例如：一個使用者可以評論多篇文章，同時一篇文章也可以被許多使用者評論。

    在 Rails 中，建立 Model 之間的關聯性通常使用前言提到的六種方法來定義。
    透過這些方法，你可以在 Model 類別中指定關聯的類別和條件，
    Rails 將根據這些定義在資料庫層面建立對應的關聯。
