Viiisit tailwind - Installation !

Zinni Chang
2023-07-15
Keep Learning
Tailwind

before_action :mindset

在嘗試安裝 tailwind 時，發現如果對於不熟悉套件的指令或者安裝當下需要動到哪些檔案，很容易重工，因此，最好的辦法就是乖乖照做，並且要理解開發新專案可以藉由哪些套件去完成！
整理出以下自己在安裝與使用上的一些想法，不過這需要每天都研究一些，所以目前只是半成品的整理資訊，離真正完整了解可能還需要一些時間！加油啊～！

tailwind installation

Step 01 先從一個全新專案開始，建立一個資料夾將之後要安裝 tailwind 的相關檔案一併放入在裡面。

  • Terminal

      Remark:
  執行 npm init -y 命令時，npm 將以預設值自動初始化一個 package.json 檔案，而不需要進行互動式的問題回答過程。
  參數 -y 是 --yes 的縮寫形式，表示自動接受預設值，非常適用於快速初始化專案或在構建自動化腳本時使用。
  如果需要自定義需求設定專案的名稱、版本、描述或者需要選擇特定的配置選項，建議使用 npm init 進行手動初始化，並根據提示回答相關問題。

Step 02: 跟著 tailwind 官網上的 Documentation

  • Terminal

      Remark:
  1. 執行 npm install -D tailwindcss 該套件將被添加到 package.json 檔案中的 devDependencies 部分，並在項目的構建和開發過程中使用。
  好處是，它只會在開發環境中使用，而不會包含在生產構建中。這樣可以確保僅在開發和構建項目時使用 tailwindcss。
  2. 執行 npx tailwindcss init 命令後，會查找本地安裝的 tailwindcss 套件，並執行它的 init 命令。
  這個命令將生成一個初始的 tailwind.config.js 配置文件，該文件用於自定義和設置 Tailwind CSS 的各種選項，包括顏色、字體、間距、斷點等等。

Step 03: 跟著 tailwind 官網上的 Documentation

  • VSCode
    • open tailwind.config.js file

Step 04: 跟著 tailwind 官網上的 Documentation

  • VSCode

    • 建立 src 資料夾，裡面新增 input.css 與 index.html

      src/input.css

Step 05: 跟著 tailwind 官網上的 Documentation

  • Terminal

      Remark:
  執行 npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch 命令後，
  tailwindcss 將讀取指定的輸入檔案（./src/input.css），根據 Tailwind CSS 的配置生成對應的 CSS 樣式，
  並將結果寫入輸出檔案（./dist/output.css）。同時，tailwindcss 會持續監聽輸入檔案的變化，並在有變更時重新編譯並更新輸出檔案。

Step 06: 跟著 tailwind 官網上的 Documentation

  • VSCode

    • src 資料夾新增 index.html

      src/index.html
